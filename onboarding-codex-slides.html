What is Codex

Codex 란?

코드 작성 · 분석 · 수정 · 리뷰 · 디버깅 · 문서화까지
수행하는 AI 코딩 에이전트
  • 코드 실행 환경과 문서 규칙을 함께 읽고 동작
  • 규칙 기반으로 작업을 수행 — 임의 판단 최소화
  • 이 레포의 모든 규칙·스킬·문서가 Codex의 행동 근거
Codex의 역할과 동작 방식 소개.

Top-Level Config

Git Worktree

워크트리란?

동일한 Git 저장소에서 여러 작업 디렉터리를 병렬로 만드는 기능
브랜치/컨텍스트를 분리해 충돌 없이 동시 작업 가능
이 환경의 워크트리 우선순위
  • 1순위 — .worktrees 존재 시 최우선 사용
  • 2순위 — worktrees 존재 시
  • 3순위 — CLAUDE 규칙/문서에서 지정된 경로
  • 4순위 — 둘 다 없으면 사용자 확인
워크트리 개념과 우선순위.

Practical Checklist

실무 체크리스트

  • Codex 요청 전: 어떤 규칙 파일이 적용되는지 확인했는가
  • 요청이 여러 작업을 담고 있는가 — Skill이 필요한가
  • 워크트리 사용이 필요한가
  • 멀티에이전트가 유리한 독립 태스크인가
  • 결과물에 근거 문서 링크를 남겼는가
    (AGENTS / rules / skill / docs)
5가지 실무 체크리스트.

FAQ

자주 묻는 질문

Q. Skills가 항상 로딩되나?
아님. 요청 조건이 맞을 때만 로드한다.
Q. Skills만 보면 되나?
아님. AGENTS + rules + docs가 함께 기준을 구성한다.
Q. 멀티에이전트가 무조건 빠른가?
독립 작업에서만 유리하며, 의존 작업에서는 역효과가 날 수 있다.
AGENTS.md 우선 Skills 조건부 로드 독립 → 병렬
