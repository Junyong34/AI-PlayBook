# AI Tool Catalog: MCP / Skill / Plugin

이 문서는 실제 작업에서 유용하게 사용하는 AI 도구를 한 곳에 정리하기 위한 운영용 카탈로그입니다.
단순 링크 모음이 아니라, 각 도구의 특징과 사용법을 빠르게 확인하고 재사용할 수 있도록 구성합니다.

## 문서 목적

- 현재 사용 중이거나 바로 운영에 투입할 수 있는 MCP, Skill, Plugin을 관리합니다.
- 각 도구의 핵심 특징, 사용 목적, 기본 사용법을 짧게 정리합니다.
- 도구별 지원 AI와 도입 상태를 기록해 유지보수 기준으로 활용합니다.

## 분류 기준

| 구분 | 의미 |
| --- | --- |
| MCP | 외부 도구, 메모리, 코드 탐색 기능을 AI 클라이언트에 연결하는 서버 |
| Skill | 특정 작업 방식을 지침화한 실행 규칙 또는 워크플로우 |
| Plugin | 명령어, Skill, MCP, CLI 등을 묶어 제공하는 확장 패키지 |
| CLI | 터미널에서 직접 실행해 리포트, 분석, 자동화를 수행하는 도구 |

## 상태 기준

| 상태 | 의미 |
| --- | --- |
| `active` | 현재 사용 중이며 운영에 활용 가능 |
| `experimental` | 평가 중이며 사용 범위가 제한적 |
| `deprecated` | 더 이상 사용하지 않거나 대체 예정 |

## 상세 카탈로그

### Session Report

| 항목 | 내용 |
| --- | --- |
| 유형 | Plugin |
| 지원 AI | both |
| 상태 | `active` |

**특징**

- 프로젝트, Skill, 프롬프트 단위의 토큰 사용량을 시각화합니다.
- 기간별 사용량을 확인해 과다 사용 구간을 찾기 쉽습니다.
- 세션 운영 비용을 정기적으로 점검할 때 유용합니다.

**설명**

AI 작업량과 토큰 소비 패턴을 리포트 형태로 확인하는 도구입니다. 장기 세션, 대형 리팩토링, 문서 작업처럼 컨텍스트가 커지는 작업에서 비용과 효율을 점검하는 데 사용합니다.

**사용법**

- `/session-report --30d`로 최근 30일 사용량을 확인합니다.
- 자연어로 “최근 토큰 사용량 분석해줘”처럼 요청해 리포트를 생성합니다.
- 비용이 비정상적으로 큰 세션은 프롬프트, Skill, 작업 단위로 원인을 나눠 확인합니다.

### Claude MD Management

| 항목 | 내용 |
| --- | --- |
| 유형 | Skill Set |
| 지원 AI | claude-only |
| 상태 | `active` |

**특징**

- `CLAUDE.md` 규칙을 정리하고 중복되거나 모호한 지침을 개선합니다.
- 프로젝트 규칙이 누적되면서 생기는 충돌을 점검하기 좋습니다.
- `revise-claude-md`, `claude-md-improver` 계열 명령을 활용합니다.

**설명**

Claude 기반 프로젝트 규칙 파일을 관리하는 Skill Set입니다. 오래된 규칙, 충돌하는 규칙, 너무 장황한 지침을 정리해 에이전트가 더 일관되게 동작하도록 돕습니다.

**사용법**

- `CLAUDE.md`를 수정하기 전 현재 규칙의 중복과 충돌을 점검합니다.
- `revise-claude-md`로 구조를 정리합니다.
- `claude-md-improver`로 문장과 실행 지침을 개선합니다.

### Serena MCP

| 항목 | 내용 |
| --- | --- |
| 유형 | MCP |
| 지원 AI | both |
| 상태 | `active` |

**특징**

- 의미 기반 코드 검색과 탐색을 지원합니다.
- 단순 문자열 검색보다 코드 구조와 맥락을 파악하는 데 강점이 있습니다.
- 큰 코드베이스에서 관련 함수, 클래스, 호출 흐름을 찾을 때 유용합니다.

**설명**

AI 클라이언트에 코드 이해용 MCP 서버를 연결해 코드 탐색 품질을 높이는 도구입니다. 파일명을 모르는 상태에서 기능의 위치를 찾거나, 특정 동작이 어디서 구현되는지 추적할 때 사용합니다.

**사용법**

- `uv` 기반으로 Serena MCP 서버를 연결합니다.
- 코드 탐색이 필요한 작업에서 MCP 호출을 통해 관련 심볼과 파일을 찾습니다.
- 구조를 파악한 뒤 실제 수정은 로컬 파일 편집 흐름으로 진행합니다.

### Hookify

| 항목 | 내용 |
| --- | --- |
| 유형 | Plugin |
| 지원 AI | both |
| 상태 | `active` |

**특징**

- 반복 실수 패턴을 등록해 작업 전에 차단합니다.
- 팀 또는 개인의 자주 발생하는 실수를 규칙화할 수 있습니다.
- 리뷰 이전 단계에서 품질 게이트 역할을 합니다.

**설명**

자주 반복되는 실수나 금지 패턴을 hook 형태로 관리하는 도구입니다. 예를 들어 특정 명령 실행 금지, 파일 수정 전 체크리스트 확인, 규칙 누락 방지 같은 흐름에 사용할 수 있습니다.

**사용법**

- `hookify`로 반복 실수 패턴을 등록합니다.
- 프로젝트별 금지 명령, 필수 확인 항목, 리뷰 전 체크 항목을 설정합니다.
- 작업 전후로 hook이 의도대로 동작하는지 확인하고 필요하면 패턴을 갱신합니다.

### codeburn

| 항목 | 내용 |
| --- | --- |
| 유형 | Plugin/CLI |
| 지원 AI | both |
| 상태 | `active` |

**특징**

- 토큰 사용량과 세션 비용을 CLI 기반으로 분석합니다.
- 비용이 큰 세션과 비효율적인 작업 흐름을 찾는 데 유용합니다.

**사용법**

- 비정상적으로 큰 세션은 작업 범위, 프롬프트 길이, 반복 읽기 여부를 점검합니다.
- 주기적으로 리포트를 저장해 사용량 추이를 비교합니다.

### andrej-karpathy-skills

| 항목 | 내용 |
| --- | --- |
| 유형 | Skill Pack |
| 지원 AI | claude-only |
| 상태 | `active` |

**특징**

- Karpathy식 코딩 원칙을 Skill 형태로 적용합니다.
- 단순성, 목표 중심 실행, 빠른 실험을 강조합니다.
- 과도한 추상화나 장황한 설계를 줄이는 데 도움이 됩니다.

**설명**

코드를 더 단순하고 직접적으로 작성하도록 유도하는 Skill Pack입니다. 복잡한 설계보다 작게 만들고 빠르게 검증하는 방식이 필요한 작업에 적합합니다.

**사용법**

- Plugin 또는 Skill Pack을 설치합니다.
- 구현 전 “단순한 접근 우선” 원칙을 적용할 작업에 활성화합니다.
- 리뷰 시 불필요한 추상화, 과한 일반화, 목표와 무관한 코드를 점검합니다.

### supermemory

| 항목 | 내용 |
| --- | --- |
| 유형 | MCP |
| 지원 AI | both |
| 상태 | `active` |

**특징**

- 메모리와 컨텍스트 기반 조회 기능을 제공합니다.
- 이전 결정, 사용자 선호, 프로젝트 맥락을 다시 불러올 수 있습니다.
- 장기 프로젝트에서 컨텍스트 손실을 줄이는 데 유용합니다.

**설명**

AI 작업 중 축적된 메모나 컨텍스트를 조회하기 위한 MCP입니다. 반복적으로 설명해야 하는 프로젝트 배경, 운영 규칙, 의사결정 기록을 재사용할 때 적합합니다.

**사용법**

- MCP 서버를 연결합니다.
- `memory`, `recall`, `context` 계열 기능으로 필요한 정보를 조회합니다.
- 중요한 결정이나 반복 참조 정보는 메모리로 저장해 다음 세션에서 재활용합니다.

### gstack

| 항목 | 내용 |
| --- | --- |
| 유형 | Skill Pack |
| 지원 AI | both |
| 상태 | `active` |

**특징**

- `autoplan`, `ship`, `investigate`, `review`, `test` 등 여러 작업 Skill을 제공합니다.
- 개발 전 과정의 반복 워크플로우를 Skill 단위로 표준화합니다.
- 계획 수립부터 검증까지 일관된 흐름을 만들기 좋습니다.

**설명**

엔지니어링 작업을 단계별 Skill로 나눠 실행하는 Skill Pack입니다. 작업 계획, 조사, 구현, 리뷰, 테스트처럼 반복되는 개발 흐름을 빠르게 호출할 수 있습니다.

**사용법**

- 사용하는 AI 클라이언트 기준으로 Skill Pack을 설치합니다.
- 작업 유형에 맞게 `autoplan`, `investigate`, `review`, `test` 등을 선택합니다.
- 프로젝트 규칙과 충돌하는 지침이 있는지 확인한 뒤 적용합니다.

### compound-engineering-plugin

| 항목 | 내용 |
| --- | --- |
| 유형 | Plugin + Skills |
| 지원 AI | both |
| 상태 | `active` |

**특징**

- `/ce-*` 계열 명령과 Skill을 제공합니다.
- 아이디어 발산, 브레인스토밍, 계획, 실행, 디버깅을 포괄합니다.
- 멀티 클라이언트 환경에서 워크플로우를 동기화하기 좋습니다.

**설명**

복합 엔지니어링 작업을 구조화하기 위한 Plugin입니다. 모호한 요구사항을 정리하거나, 계획을 작성하거나, 디버깅 흐름을 정돈할 때 사용합니다.

**사용법**

- Plugin을 설치하고 사용하는 클라이언트에 동기화합니다.
- 작업 성격에 맞게 `/ce-ideate`, `/ce-brainstorm`, `/ce-plan`, `/ce-work`, `/ce-debug` 등을 사용합니다.
- 문서화가 필요한 해결 과정은 관련 Skill을 사용해 재사용 가능한 지식으로 남깁니다.

### superpowers

| 항목 | 내용 |
| --- | --- |
| 유형 | Skill Framework |
| 지원 AI | both |
| 상태 | `active` |

**특징**

- 브레인스토밍, TDD, 디버깅, 코드 리뷰, 서브에이전트 작업을 Skill로 제공합니다.

### rtk

| 항목 | 내용 |
| --- | --- |
| 유형 | Plugin/CLI Toolchain |
| 지원 AI | both |
| 상태 | `active` |

**특징**

- 세션 분석, 컨텍스트 관리, 명령 프리셋을 지원합니다.
- CLI hook과 설정을 연동해 반복 작업을 자동화할 수 있습니다.
- 에이전트 작업 환경을 운영 도구처럼 관리하는 데 적합합니다.

**설명**

AI 세션 운영과 개발자 워크플로우를 보조하는 Toolchain입니다. 반복 명령, 컨텍스트 정리, 세션 분석처럼 운영 효율에 영향을 주는 작업을 관리합니다.

**사용법**

- CLI hook과 프로젝트 설정을 연동합니다.
- 자주 쓰는 명령을 프리셋으로 관리합니다.
- 세션 분석 결과를 보고 컨텍스트 구성과 작업 단위를 조정합니다.

## 운영 원칙

- `active` 항목은 분기별로 실제 사용 여부와 지원 AI 범위를 재확인합니다.
- `MCP`는 연결 장애나 버전 변경이 발생하면 먼저 동작 검증 후 상태를 갱신합니다.
- `Skill Pack`은 기존 규칙 파일과 충돌하지 않는지 확인한 뒤 적용합니다.
- `Plugin/CLI`는 설치 방식, 주요 명령, 리포트 위치를 별도로 기록합니다.
- 더 이상 사용하지 않는 항목은 삭제하지 말고 `deprecated`로 바꾼 뒤 대체 도구를 남깁니다.
